Репозиторий посвящен альтернативному экзамену по предмету МЛИТА, постановка задачи:


*Булевы игры используются для моделирования многих комбинаторных задач, задач планирования и т. д. Цель «Верификатора» (V) — подобрать значения переменных, чтобы сделать заданную пропозициональную формулу истинной, а цель «Фальсификатора» (F) — сделать её ложной. Обычно каждый игрок контролирует некоторый фиксированный набор переменных. Однако можно рассмотреть и случай, когда игроки по очереди выбирают переменные (ещё не используемые) и присваивают им значения. Они могут знать (или не знать) предыдущие значения, выбранные противником. В случае выполнимых формул, т. е. истинных для некоторых значений переменных, и ложных для других, тем не менее один из игроков в некоторых случаях может иметь выигрышную стратегию. Например, если A зависит от 2n переменных, а количество моделей (булевых векторов, делающих A истинным) меньше 2^n, при этом каждый игрок по очереди выбирает одну переменную, F всегда может выиграть. Мы предлагаем изучать «граничные случаи», когда число моделей равно ровно 2^n или чуть больше, чтобы получить оценку гарантированного выигрыша для F (можно показать на примерах, что гарантированный выигрыш для F сохраняется при числе моделей, близком, но все же большем, чем 2^n). Мы также предлагаем использовать систему TouIST, разработанную в IRIT, для проверки конкретных примеров, допускающих обобщение. Подобные задачи важны и для семантических игр (метод проверки истинности формул при помощи игры между V и F).*

Что фактически сделано:

- Написана программа, которая перебирает все булевы функции от 4 переменных, фиксируя какой из игроков, при оптимальной стратегии, может на ней побеждать. С помощью этой программы были получены точные оценки количества выполняющих наборов у функции, при котором F может всегда побеждать.
- Оценки, полученные программой подтверждены аналитически.
- Написана небольшая заметка, в которой более подробно изложены результаты проделанной работы.
